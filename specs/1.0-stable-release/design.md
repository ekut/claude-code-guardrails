# Design: Stable Release

## Overview

v1.0 prepares the plugin for public release: clean rule templates separated from project copies, a `/rules-setup` skill for interactive installation, complete plugin metadata, comprehensive documentation (README Installation, CONTRIBUTING.md), dogfooding validation, and a "Future Ideas" section inviting community input.

## Detailed Design

### 1. Rule Templates Directory

Create a `rules/` directory at the project root with clean, unmutated rule templates:

```
rules/
  spec-driven-design.md
  search-before-build.md
  testing-discipline.md
  pr-quality.md
  security-practices.md
  documentation-standards.md    # base only, without CHANGELOG format preference
```

`git-workflow.md` is NOT included — it is generated by `/git-flow-setup` with the user's choices.

Our project's `.claude/rules/` remains a real directory with our mutated copies. This is already the case.

### 2. Skill: `rules-setup`

A new skill at `skills/rules-setup/SKILL.md` invoked via `/rules-setup`.

**Flow:**
1. Check if `.claude/rules/` exists — create if needed
2. Scan `rules/` directory for available rule templates
3. For each template, read the first line (title) and show a list with install status:
   - Already installed (file exists in `.claude/rules/`)
   - Not installed
4. Let the user select which rules to install (multi-select)
5. For each selected rule:
   - If already exists in `.claude/rules/` — ask: "Rule already exists. Overwrite or skip?"
   - Copy the clean template to `.claude/rules/{name}.md`
6. After copying, suggest next steps:
   - "Run `/git-flow-setup` to configure git workflow rules"
   - "Run `/hooks-setup` to install git hooks"
7. Report what was installed

### 4. README Installation Section

Replace the current placeholder with clear instructions for three audiences:

**Plugin installation** (recommended):
```
/plugin marketplace add ekut/claude-code-guardrails
/plugin install claude-code-guardrails
```
Then run `/rules-setup`, `/git-flow-setup`, `/hooks-setup`.

**Manual installation** (advanced users):
- Clone/submodule the repo
- Cherry-pick specific skills, agents, or rules you need
- Copy to your project's `.claude/` directory

**Update:**
```
/plugin marketplace update
```

**Uninstall:**
```
/plugin uninstall claude-code-guardrails
```

## Alternatives Considered

| Alternative                        | Pros            | Cons                       | Why Rejected                                                         |
|------------------------------------|-----------------|----------------------------|----------------------------------------------------------------------|
| Auto-install rules on plugin setup | Zero friction   | Modifies project without ask | User should opt-in via `/rules-setup`                              |
| Rules as CLAUDE.md content         | No copy needed  | Can't be selective         | Users want to pick which rules to adopt                              |
| Single setup wizard for everything | One command      | Too long, overwhelming     | Separate skills (rules-setup, git-flow-setup, hooks-setup) are more focused |

## Risks and Unknowns

- Plugin installation UX depends on Claude Code's plugin system stability — we document the current approach and note it may evolve
- Some users may want to customize rules after installation — rules are plain markdown files, so editing is straightforward

## Dependencies

- All v0.1–v0.8 milestones (complete)
- Claude Code plugin system (for installation)
